# 1 helix配置
## 1.1 设置为默认编辑器
通过bash配置文件(~/.bashrc)配置helix为默认编辑器
	`export EDITOR=helix`
	`export VISUAL=helix`


## 1.2 创建用户配置文件
	`$ touch ~/.config/helix/config.toml`
	`$ touch ~/.config/helix/languages.toml`


## 1.3 基础配置
进入~/.config/helix/config.toml文件
	1. 主题选择
		`theme = "autumn"`
	2. 鼠标形状配置
		`[editor.cursor-shape]
		        insert = "bar"
			normal = "block"
			select = "underline"`
	3. 文件选择器配置
		`[editor.file-picker]
			hidden = false`
	4. 状态栏配置
		`[editor.statusline]
			left = ["mode", "spinner"]
			center = ["file-name"]
			right = ["diagnostics", "selections", "position", "file-encoding", "file-line-ending", "file-type"]
			separator = "│"
			mode.normal = "NORMAL"
			mode.insert = "INSERT"
			mode.select = "SELECT"
			diagnostics = ["warning", "error"]
			workspace-diagnostics = ["warning", "error"]`
	5. 按键映射配置
		`[keys.normal]
			C-s = ":w"
			C-x = ":q!"
			C-q = ":wq"
			C-down = "half_page_down"
			C-up = "half_page_up"`


## 1.4 lsp语言配置
进入~/.config/helix/language.toml文件
	1. Bash语言配置
		`[[language]]
			name = "bash"

# 2 helix使用
## 2.1 主要模式
	1. normal模式： 用于文件的导航和使用命令行，使用"Esc"键从insert,select模式进入
	2. insert模式： 用于文件的编辑，使用“i”键从normal,insert模式进入
	3. select模式： 用于文件的选择，使用“v”键从normal模式进入


## 2.2 选择文件
	- space+e: 打开工作区目录
	- space+E: 打开当前文件所在目录
	- space+f: 从工作区的所有文件中选择
	- space+F: 从当前文件所在目录的文件中选择
	- space+b: 从缓存区的文件中选择
	- space+g: 从工作区的被修改文件（git仓库）中选择


## 2.3 文件内容导航
	1. 移动当前视口
		- z: 展示功能列表
		- 使用绑定的快捷键		
	2. 跳转到文件其他部分
		- g: 展示跳转功能列表
			- d: 跳转到定义处
			- D: 跳转到申明处
			- y: 跳转到数据类型定义处
			- r: 列出所有引用
			- i: 跳转到实现处
	3. 搜索与移动
		- /: 展示输入框，输入内容对当前文件搜索
		- space+/: 展示输入框，输入内容对工作区所有文件搜索 
		- 在获取搜索结果后，使用“n”跳转到下一个匹配项，“N”跳转到上一个匹配项
	4. 基于lsp的符号选择跳转
		- space+s: 从当前文件的所有符号（函数，类，变量）中选择
		- space+S: 从工作区所有文件的符号中选择
	5. 基于Lsp的诊断信息
		- space+d: 从当前文件列出的诊断信息中选择
		- space+D: 从工作区所有文件的诊断信息中选择


## 2.4 其他功能
	- space+y: 复制选择部分到剪切板
	- space+p: 粘贴复制内容到选择部分之后
	- space+c: 注释/取消注释选择的部分
	- space+k: 显示选择部分的文档解释
	- space+w: 打开窗口编辑功能列表
