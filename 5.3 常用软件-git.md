# 2 git配置
下载安装软件：
	`$ sudo pacman -S git`

通过bash配置文件(~/.bashrc)配置命令别名
  	`alias g='git'`

通过用户的git配置文件(~/.config/git/config.ini):
配置完后可运行`git config --list --global`验证配置
	1. 配置默认用户名与邮件
  		[user]
			name = maodun
    			email = zaidao.jiang@gmail.com
	2. 对**git init**命令配置：修改初始化仓库的默认名称
  		[init]
    			defaultBranch = main
	3. 对**git status**命令配置: 显示仓库的分支信息和暂存区信息
		[status]
    			branch = true
    			showStash = true
	4. 对**git push**命令配置: 远程与本地仓库分支自动对应,默认推送当前分支内容,标签可以一同推送 
  		[push]
    			autoSetupRemote = true
    			default = current
	5. 对**git pull**命令配置: 默认拉取当前分支内容
  		[pull]
			default = current

通过创建并编辑仓库中的".gitignore"文件,列出要忽略的文件的格式或名字
可以手动编辑或者通过工具为具体项目自动生成,如github在建立仓库时会有相应选项       



# 3 本地仓库命令
## 3.1 初始化本地仓库
使用**git init**命令本地目录初始化为本地仓库  
	`$ git init`


## 3.2 获取仓库与文件信息
使用**git status**命令获取仓库状态 
	`$ git status`

使用**git diff**命令比较文件间的差异
	- `$ git diff`: 比较工作区与暂存区文件的差异  
	- `$ git diff --staged`: 比较暂存区文件与最新提交的差异  
	- `$ git diff commit1 commit2`: 比较不同提交间的差异  

使用**git log**命令按时间顺序列出所有提交
	`$ git log`


## 3.3 本地文件控制
使用**git add**命令追踪文件并存放暂存区  
	`$ git add filename`
  		- "filename": 选择特定文件
  		- "-A": 选择所有文件  

使用**git commit**命令将暂存区文件提交形成一次快照
	`$ git commit filename`
  		- "-m content": 编辑此次提交的注释
  		- "--amend": 追加提交到最新一次提交的快照

使用**git restore**命令恢复文件  
	- `$ git restore filename`: 丢弃该文件在工作区时的修改
	- `$ git restore --staged filename`: 将文件从暂存区移除,但保留在工作区时的修改
	- `$ git restore --source=commit --staged --worktree filename`: 将文件从暂存区移除,丢弃在工作区时的修改,并恢复到指定commit时的内容
    		- "--source=HEAD": 最新的提交
    		- "--worktree": 表示丢弃在工作区时的修改

使用**git rm**命令删除文件
  	`$ git rm filename`: 删除文件并将此修改提交到暂存区
  	`$ git rm --cached filename`: 移除本地仓库对此文件的跟踪,保留本地文件
 
使用**git mv**命令重命名或移动文件/文件夹
	- `$ git mv old_filename new_filename`: 文件/文件夹重命名
	- `$ git mv filename path`: 移动文件
	- `$ git mv old_filename path/new_filename`: 移动文件并重命名



# 4 远程仓库命令  
使用**git remote**命令管理与本地仓库关联的远程仓库
	- 查询远程仓库信息
  		`$ git remote -v`: 显示简名与URL  
  		`$ git remote show shortname`: 查看某个远程仓库的详细信息  
	- 管理远程仓库
  		`$ git remote add shortname url`: 连接到到远程仓库并自定义本地对此远程仓库的简名
  		`$ git remote remove shortname`: 移除远程仓库
	- 修改远程仓库信息
  		`$ git remote rename old_shortname new_shortname`: 修改本地对某个远程仓库的简名
  		`$ git remote set-url shortname new_url`: 修改简称远程仓库的新URL  



# 5 远程与本地仓库交互命令
使用**git clone**命令克隆远程仓库为本地仓库并自定义简名,同时本地仓库连接远程仓库
	`$ git clone url shortname`

使用**git fetch**命令获取远程仓库最新提交的内容,但不合并到本地仓库
	`$ git fetch shortname branchname`: 获取远程仓库分支的最新提交的数据  

使用**git merge**命令将其他分支的变更合并到当前分支
	`$ git merge shortname/branchname`: 将获取的远程仓库分支的最新提交合并到当前分支
	`$ git merge branchname`: 将本地仓库分支合并到当前分支

使用**git pull**命令获取并合并远程仓库最新提交的内容到对应本地仓库
	`$ git pull`: 默认拉取与当前仓库分支对应的远程仓库分支
	`$ git pull shortname branchname`: 获取并合并远程仓库分支的最新提交的内容到对应本地仓库

使用**git push**命令将本地仓库分支推送到远程仓库分支，如果远程仓库没有该分支则会在远程仓库触发“pull request”请求并创建相应分支
	`$ git push shortname branchname`: 将本地仓库分支推送到远程仓库对应分支
	`$ git push shortname local_branchname:remote_branchname`: 将本地仓库分支推送到远程仓库分支
	`$ git push shortname HEAD`: 将本地仓库当前分支推送到远程仓库对应分支
	`$ git push shortname --delete branchname`: 删除远程仓库的分支   

> **推送前先获取与合并,避免冲突**


  
# 6 仓库分支命令  
使用**git branch**命令对分支查看,创建与删除
	- 查询分支信息
  		`$ git branch`: 查询当前分支信息  
    			- "-v": 查看当前分支与其最新提交信息
    			- "-a": 查看全部分支
	- 管理分支  
  		`$ git branch branchname`: 创建新分支,不切换到新分支  
  		`$ git branch -d`: 删除分支,需要切换出该分支
	- 修改分支信息 
  		`$ git branch -m old_branchname new_branchname`: 重命名当前分支

使用**git switch**命令切换分支
	`$ git switch branchname`: 切换分支
