# 1 C语言
## 1.1 配置C语言开发环境
	1. 下载软件
		`$ sudo pacman -S clang`
	2. 验证安装情况
		`$ clang --version`


## 1.2 使用**clang**命令编译文件
	1. 只进行预处理
		`$ clang -E filename.c -o filename.i`
	2. 只进行到编译，生成汇编代码文件
		`$ clang -S filename.c -o filename.s`
	3. 只进行到汇编，生成机器码文件
		`$ clang -c filename.c -o filename.o`
	4. 执行完整流程到链接，生成可执行文件
		`$ clang filename.c -o programename
	5. 多文件编译
		- 将多文件编译至机器码文件阶段，再将多个机器码文件链接起来
			`$ clang file1.o file2.o -o programename`
		- 链接其他库一同编译
			1. 链接库
				`$ clang filename.c -L/path -l<libraryname>`
			2. 静态链接
			3. 动态链接
	6. 编译过程中其他选项
		- -o: 自定义生成的文件名称 
		- -std=<standard>: 指定语言标准
		- -g: 生成调试信息
		- v: 显示详细过程
		- -Ox：进行优化，x为优化等级，建议“O2”
		- wx: 警告控制，x为警告等级，建议“wall”

# 2 Rust语言
## 2.1 配置Rust语言开发环境
下载rustup工具链管理器
	- 通过archlinux仓库下载
		`sudo pacman -S rustup`
	- 通过rust官方命令下载，选择默认选项
		`$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh`

检查相关命令是否在环境路径变量下
	- 如果是从仓库下载，默认命令会放在`/usr/bin`目录下
	- 如果是官方命令下载，会在“.bash_profile"中写入以下内容以自动执行脚本，添加目录`~/.cargo/bin`到环境变量
		`. "$HOME/.cargo/env"`

安装相关工具链
	- 如果是从仓库下载，需要手动安装工具链条
		`$ rustup default stable`
			- stable: 稳定版
			- beta: 测试版
			- nightly: 开发版
	- 如果是官方命令下载，在下载过程中默认安装stable版本工具链

安装linker链接器,可以选择安装Clang编译器,其包含链接器
	`$ sudo pacman -S clang`

测试rustup是否安装成功
	`$ rustc -V`

安装rust语言lsp
	`$ sudo pacman -S rust-analyzer`

> **建议使用archlinux仓库下载rustup**


## 2.2 使用**rustup**命令
	1. 更新rustup的命令（官方命令下载） 
		`$ rustup self update`
	2. 下载与切换工具链的命令
		`$ rustup default stable`
	3. 更新工具链的命令
		`$ rustup update`
	2. 卸载rustup的命令（官方命令下载）
		`$ rustup self uninstall`
	3. 查看文档
		`$ rustup doc`


# 2.3 使用**Cargo**命令
	1. 检查cargo版本
		`$ cargo --version`
	2. 创建新项目
		- `$ cargo new projectname`
		- 会生成名为“Projectname”的新目录,新目录会生成以下文件:
			- src文件夹
				- 存放源代码的文件夹
				- 里面包含一个"main.rs"文件
			- cargo.toml
				- [package]用来配装项目基本信息
					- name: 项目名称
					- version: 项目的版本
					- edition: 使用的Rust版本
				- [dependencies]是罗列项目所依赖得代码包crates
					- 库名称:"版本号"
	3. 将原项目转换为使用cargo的项目
		`$ cargo init`
	4. 构建项目
		- `$ cargo build`
		- 会生成以下重要文件
			- target文件夹
				- 存放构建出的项目
			- target/debug文件夹
				- 默认的构建方法是调试构建(debug build)，所以创建debug文件夹
				- 存放调试构建出的项目，可执行文件在其中
			- Cargo.lock
				- 记录项目依赖的实际版本

		- 可以通过这个命令可执行文件目录下运行
			`$ ./filename`

	5. 构建并运行项目
		`$ cargo run`
	6. 测试项目
		- 在不生成二进制文件的情况下构建项目来检查错误
			`$ cargo check`
		- 执行完整全面的项目测试，生成二进制文件
			`$ cargo test`
		- 当在写代码过程中时，频繁使用`cargo check`来检查
		- 当功能完成时，使用`cargo test`来验证
	7. 生成项目文档
		`$ cargo doc`
	8. 发布构建项目 
		- 当准备发布时,为了优化编译项目让其运行更快
		- `$ cargo build --release`
		- 会生成以下重要文件
			- target/release文件
				- 该方法是发布构建(release build)，所以创建release文件夹
				- 存放发布构建出的项目，可执行文件在其中
	9. 将库发布到“crato.io”(社区库)
		- `$ cargo publish`
